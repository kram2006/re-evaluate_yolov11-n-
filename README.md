# YOLOv11n Repository

This repository contains a YOLO-based model (`yolo11n.pt`) and helper scripts for validating and running detection on the COCO 2017 dataset. The dataset currently stored in this workspace is large (~56–57 GB). This README documents the project layout, how to run validation, and safe ways to remove or archive the dataset if you want to free disk space.

## Repository layout
- `validate_yolov11n.py` — validation / evaluation script used with the included model.
- `yolo11n.pt` — pretrained model weights file (present in repo root).
- `datasets/coco/` — COCO 2017 dataset (images, labels, annotations, and split lists). This is the large directory you may want to remove or archive.
- `runs/detect/` — output folders from detection/validation runs. Example: `runs/detect/val/predictions.json`.
- `yolo_env/` — a Python virtual environment that was used to run the code.

## Dataset details (IMPORTANT)
- Location: `datasets/coco/`.
- Approximate size on disk: 56–57 GB (contains images and label data for COCO 2017).
- Contents:
  - `images/train2017/` — training images (many GBs).
  - `images/val2017/` — validation images.
  - `images/test2017/` — test images (if present).
  - `labels/` — label files matching images (if dataset was prepared for training).
  - `annotations/instances_val2017.json` — COCO-style JSON annotations used for evaluation.
  - split lists: `train2017.txt`, `val2017.txt`, `test-dev2017.txt` — text files with image ids/paths used by scripts.

Before deleting or removing this directory, ensure you have backed up any files you may need later (annotations, custom labels, or lists). The evaluation process requires the `annotations/instances_val2017.json` file and either the images or precomputed predictions.

## Why you can delete the dataset
- If your goal is only to keep the model (`yolo11n.pt`) and previously computed predictions, you can safely delete the full image dataset after making sure:
  - You have copies of `annotations/` (if you will re-run COCO evaluation), or
  - You keep the JSON predictions generated by prior runs (`runs/detect/*/predictions.json`) so you can re-evaluate later without images.

Note: Some evaluation scripts require the images to compute certain metrics, but COCO evaluation primarily uses the annotations and prediction JSONs.

## Safe deletion / archive options
Use one of the options below depending on what you prefer.

1) Move dataset to external drive (recommended if you might restore later)

PowerShell example:

```powershell
# Move to D:\backup\coco (create destination folder first)
Move-Item -Path .\datasets\coco -Destination D:\backup\coco -Force
```

2) Compress into a zip (may require similar free space to create the archive)

```powershell
# Creates coco.zip in the current directory
Compress-Archive -Path .\datasets\coco -DestinationPath .\coco.zip -Force
```

3) Permanently delete (ONLY if you are sure)

```powershell
# Irrecoverable deletion
Remove-Item -LiteralPath .\datasets\coco -Recurse -Force
```

Windows `rd` (legacy) alternative in cmd/powershell:

```powershell
rd /s /q datasets\coco
```

Warning: Deletion is permanent unless you have backup or Recycle Bin settings that retain large deletes. Confirm backup before removing.

## Re-downloading COCO 2017 (if needed)
Official COCO 2017 image zips and annotations are available from the COCO website. Common files:

- Train images: http://images.cocodataset.org/zips/train2017.zip
- Val images:   http://images.cocodataset.org/zips/val2017.zip
- Test images:  http://images.cocodataset.org/zips/test2017.zip
- Annotations:  http://images.cocodataset.org/annotations/annotations_trainval2017.zip

You can re-download these with `curl`/`wget` or a browser. After download, extract into `datasets/coco/` so the script paths remain consistent.

## Running validation / quick checks
1. Activate the included virtual environment (Windows PowerShell):

```powershell
.\yolo_env\Scripts\Activate.ps1
```

Or using the batch script (cmd):

```powershell
.\yolo_env\Scripts\activate.bat
```

2. Run the validation script. Example (adjust any CLI args your script expects):

```powershell
python validate_yolov11n.py
```

Notes:
- If `validate_yolov11n.py` requires a `--data` or `--weights` argument, point it to `datasets/coco` and `yolo11n.pt` respectively.
- Evaluation outputs (predictions) are stored under `runs/detect/` by default in this workspace. Example: [runs/detect/val/predictions.json](runs/detect/val/predictions.json).

## If you remove the dataset but keep predictions
- You can still compute many evaluation metrics by supplying the prediction JSON and the annotations file `annotations/instances_val2017.json` to a COCO evaluation script.

## Notes about environment and dependencies
- A Python virtual environment is included at `yolo_env/` (likely Python 3.10 based on `Lib` packages). Activate it as shown above.
- If you need to recreate dependencies, consider running `pip install -r requirements.txt` if you add a `requirements.txt`, or create one from the environment with:

```powershell
# After activating venv
pip freeze > requirements.txt
```
